## Key Features Implemented:

1. **KPI Selection Dropdown**: Each KPI field now has a dropdown to select from available agency KPIs
2. **Slug Display**: Badge shows the selected KPI slug (e.g., "outbound_calls") for submission mapping
3. **Preselection**: Form builder loads scorecard_rules.selected_metrics and preselects KPIs
4. **Label Override**: Users can customize the display label while keeping the underlying KPI mapping
5. **Persistent Selection**: Form schema stores selectedKpiId and selectedKpiSlug for submission processing

## Flow:
1. User opens Form Builder
2. System loads agency KPIs via `list_agency_kpis(agency_id)`
3. System loads scorecard rules for the selected role
4. Form preselects KPIs based on scorecard_rules.selected_metrics  
5. User can modify selections via dropdown
6. Form schema saves KPI selections
7. On submission, values map to metric keys using selectedKpiSlug
